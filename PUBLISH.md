# Publish Checklist

## Before Publishing

1. Run full test suite:
   ```sh
   bun test --parallel
   ```

2. Run static analysis:
   ```sh
   bun run audit:static
   ```

3. Verify build:
   ```sh
   bun run build
   ```

4. Check package contents:
   ```sh
   npm publish --dry-run
   ```

5. Update version (both packages via workspaces):
   ```sh
   npm version --workspaces prerelease --preid=alpha
   # or
   npm version --workspaces patch
   ```

## Publishing

Both packages are published together via workspaces.

For alpha releases:
```sh
npm publish --access public --tag alpha
npm publish --workspace packages/ci-perf-lint --tag alpha
```

For stable releases:
```sh
npm publish --access public
npm publish --workspace packages/ci-perf-lint
```

## After Publishing

1. Verify on npm:
   ```sh
   npm info @yoshi-taka/ci-perf-lint
   npm info ci-perf-lint
   ```

2. Test install:
   ```sh
   bunx @yoshi-taka/ci-perf-lint --version
   bunx ci-perf-lint --version
   ```

3. Stable tag push creates GitHub Release automatically

4. Rebuild and verify docs site:
   ```sh
   bun run docs:build
   ```

5. Deploy docs site from `main`

## Notes

- Scoped packages default to private; always include `--access public`
- Alpha releases use `--tag alpha` so they don't become the default install
- Stable tag pushes create a GitHub Release automatically
- Tags with `-` such as `v0.0.1-alpha.10` do not auto-create a GitHub Release
- `prepack` script runs `bun run build` automatically before publish
- Docs site source of truth is `docs/rules/*.md`
- `site/src/pages/rules/*.md` is generated by `bun run docs:sync`; do not edit by hand
- Docs site build output is `site/dist`
- Current docs site build command is `bun run docs:build`
